# Tang Primer 20K Dock UART Example

This example targets the Sipeed Tang Primer 20K with the Dock extension board.

After programming the FPGA, the design transmits:

```text
Hello from Primer 20K!
```

repeatedly on the Dock board's onboard UART at `115200 8N1`.

## What This Example Assumes

- FPGA device: `GW2A-LV18PG256C8/I7`
- Board: Tang Primer 20K Dock
- Clock pin: `H11` (27 MHz)
- Reset pin: `T10`
- UART TX pin: `M11`

Those pins come from Sipeed's Tang Primer 20K documentation and example repository.

If you have the Lite baseboard instead of the Dock baseboard, you should update the constraints file before programming.

## Files

- `src/top.v`: top-level design and message scheduler
- `src/uart_tx.v`: UART transmitter
- `sim/tb_top.cpp`: Verilator testbench used for waveform generation
- `constraints/primer20k-dock.cst`: Gowin constraints for the Dock board
- `build-open-source.sh`: helper script for the open-source P&R flow
- `render-netlists.sh`: generates Graphviz and netlistsvg output
- `run-sim.sh`: builds and runs a Verilator simulation that writes a waveform file

## Verify In Docker

From the repository root:

```sh
docker compose run --rm eda-tools \
  verilator --lint-only -Wall \
  examples/primer20k-dock-uart/src/top.v \
  examples/primer20k-dock-uart/src/uart_tx.v
```

## Generate Waveforms

This example includes a Verilator simulation flow that writes an `.fst` waveform by default.

From the repository root:

```sh
docker compose run --rm eda-tools \
  sh -lc 'cd examples/primer20k-dock-uart && sh run-sim.sh'
```

This creates:

- `build/sim/top.fst`

The simulation testbench also decodes the UART stream. With the default cycle count, it now checks that the output contains:

- `Hello from Primer 20K!`
- followed by `CR` / `LF`

You can also choose the output path and simulation length:

```sh
docker compose run --rm eda-tools \
  sh -lc 'cd examples/primer20k-dock-uart && sh run-sim.sh build/sim/top.fst 120000'
```

If you want a VCD instead, pass a `.vcd` filename and the script will switch Verilator to VCD tracing automatically:

```sh
docker compose run --rm eda-tools \
  sh -lc 'cd examples/primer20k-dock-uart && sh run-sim.sh build/sim/top.vcd 120000'
```

## Generate Graphical Output

From the repository root:

```sh
docker compose run --rm eda-tools \
  sh -lc 'cd examples/primer20k-dock-uart && sh render-netlists.sh'
```

This creates:

- `build/top.dot`: Graphviz netlist from Yosys `show`
- `build/top-netlistsvg.svg`: schematic generated by `netlistsvg`

If you want to render the Graphviz file to SVG as well:

```sh
docker compose run --rm eda-tools \
  sh -lc 'cd examples/primer20k-dock-uart && dot -Tsvg build/top.dot -o build/top-graphviz.svg'
```

## Program With Gowin IDE

This is the most straightforward way to get the example onto the board with the current workspace setup.

1. Create a new Gowin project for device `GW2A-LV18PG256C8/I7`.
2. Add both Verilog files from `src/`.
3. Add `constraints/primer20k-dock.cst` as the constraints file.
4. Run `Synthesize`, then `Place & Route`.
5. Open `Programmer` and load the generated `.fs` bitstream.
6. Program SRAM for a quick test, or external flash if you want it to boot on power-up.

On the Dock board, make sure DIP switch `1` enables the core board before programming.

After programming:

1. Connect the board's USB serial interface to your computer.
2. Open a serial terminal at `115200`, `8 data bits`, `no parity`, `1 stop bit`.
3. You should see the hello message repeating once per second.

## Open-Source Build Flow

The Docker image now includes the tools needed for the open-source Gowin flow:

- `nextpnr-himbaechel`
- `gowin_pack`
- `openFPGALoader`

Build a bitstream with:

```sh
cd examples/primer20k-dock-uart
sh build-open-source.sh
```

Program it with:

```sh
openFPGALoader -b tangprimer20k build/primer20k-dock-uart.fs
```

The open-source flow uses:

- `--device GW2A-LV18PG256C8/I7`
- `--vopt family=GW2A-18`
- `gowin_pack -d GW2A-18`

With the current Docker image, Yosys comes from Debian Bookworm (`0.23`), so `build-open-source.sh` intentionally uses:

```sh
synth_gowin -top top -json build/top.json
```

and does not pass `-family`, because that option is not available in this packaged Yosys version.

## Notes

- The UART baud divisor uses a rounded divider for the 27 MHz system clock. The resulting baud rate is still well within normal UART tolerance for `115200`.
- `reset_n` is active-low and uses the documented reset pin for the Dock board.
